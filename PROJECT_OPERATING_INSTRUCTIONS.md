# BridgeStay Project Operating Instructions

## Project Summary

BridgeStay is a startup project focused on housing solutions for international students. This workspace includes product development, operations, research, legal materials, and company records.

## Main Objective

Turn this workspace into a clean, scalable, professional system that supports:
- product development
- business planning
- research
- legal documentation
- company record management

## Folder Rules

- `app/` = product and software code only
- `docs/` = project documentation, internal writeups, planning docs
- `research/` = market research, housing research, user research, raw notes
- `scripts/` = Python scripts, pipelines, automation tools
- `legal/` = platform legal docs, policies, agreements, lease docs
- `company/` = official BridgeStay LLC company records and compliance files
- `archive/` = outdated, duplicate, export, or deprecated materials

## Company File Rules

All official company records should live under: `company/BridgeStay LLC/`

Suggested subfolders:
- `formation/`
- `filings/`
- `tax/`
- `compliance/`
- `agreements/`
- `records/`
- `templates/`

## Operating Rules for Claude

When working on this project:

1. Audit before major restructuring.
2. Recommend moves before deleting or merging files.
3. Preserve originals for sensitive or legal documents.
4. Keep naming conventions clear and consistent.
5. Separate startup/company records from app development materials.
6. Prefer founder-friendly simplicity over enterprise complexity.

## Preferred Workflow

1. Identify what exists.
2. Propose improved structure.
3. Reorganize step by step.
4. Create README/index files where useful.
5. Archive outdated or duplicate materials safely.

## Naming Preference

Use clear, human-readable names. Prefer:
- lowercase-with-dashes for folders/files when possible
- descriptive file names
- avoid random exports in main root folder

## Priority

Highest priority:
1. legal/company document organization
2. docs/research cleanup
3. scripts cleanup
4. archive cleanup
5. app documentation
